# Project card conventions

Rules for the project cards in `index.html` (`.work__grid`). Apply consistently to every card and to any new card added.

## Tags
- Use **"OpenAI API"** (not bare "OpenAI") when a card depends on the OpenAI API.
- When a language tag such as **Python** is included, it goes **last**. Lead with domain/specialty keywords (RAG, Statsmodels, Claude Code, etc.); end with the implementation language.
- 4–5 tags per card, recruiter/ATS-focused keywords.

## Descriptions
- One sentence. **No em dashes (—)** — use commas.
- Uniform shape: `<Project type> that <does X>, <continuation>.`
